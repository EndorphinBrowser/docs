43 // These two classes constitute a dirty hack around QCompleter's inflexibility:
44 // QCompleter does not allow changing the matching algorithm; it is fixed
45 // at a simple QString::startsWith() comparison against the model's completionRole()
46 // data. (QCompleter also does not allow post-facto sorting of the completion results,
47 // the source model must already be completed.) To work around these limitations,
48 // we create a custom subclass of QCompleter which abuses the QCompleter::pathFromIndex()
49 // virtual override to tell the HistoryCompletionModel which string to look for,
50 // _before_ the QCompleter tries to do matching. Since the HistoryCompletionModel does
51 // its own filtering, we just lie to the QCompleter and tell it everything matches. We then
52 // abuse QCompleter::pathFromIndex() to return a url that does not start with what
53 // the user typed -- but is what they were looking for.
